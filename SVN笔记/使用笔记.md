# SVN 使用笔记（常用命令）

## 1. 仓库基础

```bash
svn checkout <repo-url> <wc-path>
svn co <repo-url> <wc-path>         # checkout 的简写

svn status

# 新增单个文件
svn add <local-file>
svn commit <local-file> -m "xxx"

# 新增当前目录内文件/目录
svn status                          # 先确认新增内容
svn add --force .
svn commit . -m "xxx"
```

## 2. 差异与日志

```bash
svn diff
svn diff <local-path>

svn log
svn log -l 10
svn info
svn blame <local-path>
```

## 3. 分支

```bash
# 创建远端分支（URL 操作会直接提交）
svn copy <trunk-url> <branch-url> -m "create branch"

# 切换当前工作副本到分支
svn switch <branch-url> .

# 合并分支到当前工作副本
svn merge --dry-run <branch-url> .
svn merge <branch-url> .
svn commit . -m "merge branch"

# 删除远端分支（URL 操作会直接提交，谨慎）
svn delete <branch-url> -m "delete branch"
```

## 4. 远端与同步

```bash
svn update
svn up
```

## 5. 撤销、回滚与冲突（谨慎）

```bash
svn status
svn diff

svn revert <local-path>             # 丢弃工作区修改，不可恢复
svn revert -R .                     # 递归丢弃全部工作区修改，不可恢复

# source-url 是该版本所在分支/路径
svn merge -c -<revision> <source-url> .
svn commit . -m "revert r<revision>"

# 人工处理冲突后，标记为已解决
svn resolve --accept working <local-path>
```

## 6. 选择性提交与提交前检查

```bash
svn status
svn diff
svn commit <local-path> -m "xxx"    # 只提交指定路径
# targets-file 一行一个路径
svn commit --targets <targets-file> -m "xxx"
```

## 7. 忽略文件

```bash
# svn:ignore 只影响未版本控制的文件/目录
svn propedit svn:ignore <local-dir>
svn propset svn:ignore "<pattern>" <local-dir>
svn propget svn:ignore <local-dir>
svn commit <local-dir> -m "update ignore"
```

## 8. 标签（tag）

```bash
# 创建远端标签（URL 操作会直接提交）
svn copy <trunk-url> <tag-url> -m "v1.0.0"
svn list <tags-url>
svn delete <tag-url> -m "delete tag" # 删除远端标签，谨慎
```

## 9. 文件与目录操作

```bash
# 本地路径操作只排程，需再提交
svn mkdir <local-dir>
svn move <old-path> <new-path>
svn copy <source-path> <target-path>
svn delete <local-path>
svn commit . -m "xxx"

# URL 操作会直接提交
svn delete <url> -m "delete path"   # 谨慎
```

## 10. 清理（谨慎）

```bash
svn cleanup                         # 清理锁/中断状态
svn status --no-ignore              # 先确认未版本控制/忽略文件
svn cleanup --remove-unversioned    # 删除未版本控制文件，不可恢复
svn cleanup --remove-ignored        # 删除忽略文件，不可恢复
```
