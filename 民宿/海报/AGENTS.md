# 民宿海报生成流程

## 资源目录

- `assets/周边环境/`：海岸、露台、航拍、建筑外观、周边氛围。
- `assets/客厅/`：客厅、石墙、原木、绿植、公共空间。
- `assets/双床房/`：双床房、窗景、阳台、房间细节。
- `assets/大床房/`：大床房、窗景、卫浴、房间细节。

## 默认流程

1. 先了解用户本轮海报目标、风格、尺寸、用途和限制。
2. 读取 `assets/` 分类，从原始素材中选取合适图片。
3. 新建递增版本目录：已有 `V1` 时新建 `V2`，依次类推。
4. 当前版本目录内放置 `assets/`、`提示词.md`、`dist/`。
5. 用子 agent 只做素材理解、视觉 brief 压缩、生成结果评审。
6. 主线程负责调用 `$imagegen`、保存图片、维护版本目录。
7. 每轮先生成 1 张主视觉，输出到当前版本的 `dist/`。
8. 子 agent 按真实感、构图、民宿感、可排版空间评分。
9. 主线程保留最佳图，每次只针对一个主要问题迭代提示词。
10. 完成一次开发后，按根目录 `COMMIT_CONVENTION.md` 进行 git 提交并推送。

## Pillow

- Pillow 安装在 `海报/.venv`，用于海报本地合成。
- 适用场景：原图裁切、拼接、渐变过渡、圆角、阴影、半透明遮罩、文字排版、导出 PNG/JPG。
- 房型、周边风景等要求使用原图时，优先用 Pillow 排版合成，不用 AI 重绘。

## HTML 海报

- 图文介绍类海报优先使用 HTML + CSS 实现。
- 当前版本目录内使用 `poster.html` 开发排版，截图输出到 `dist/`。
- HTML 适用场景：房型介绍、周边介绍、多图排版、文字层级、卡片、渐变、响应式微调。
- 默认字体使用 `PingFangSC`，不可用时使用系统默认字体。
- 开发完成后必须由子 agent 使用 `$agent-browser` 打开本地 HTML、截图调试并导出最终图片。
- 默认截图尺寸为竖版，优先保证美观度，不限制高度和大小。

## 约束

- 默认不让子 agent 直接生成图片。
- 默认先生成无字主视觉，文字后期再加。
- 每轮输出必须保存到当前版本目录的 `dist/`。
- 方向确认正确后，才考虑生成多张变体。
- 不覆盖旧图，使用递增文件名。
- 只在用户确认后进入下一轮生成。
- 每次提交只包含一个可独立理解、可独立回滚的变更。
