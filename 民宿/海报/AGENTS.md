# 民宿海报生成流程

## 资源目录

- `assets/周边环境/`：海岸、露台、航拍、建筑外观、周边氛围。
- `assets/客厅/`：客厅、石墙、原木、绿植、公共空间。
- `assets/双床房/`：双床房、窗景、阳台、房间细节。
- `assets/大床房/`：大床房、窗景、卫浴、房间细节。

## 默认流程

1. 先了解用户本轮海报目标、风格、尺寸、用途和限制。
2. 读取 `assets/` 分类，了解可用原始素材。
3. 新建递增版本目录：已有 `V1` 时新建 `V2`，依次类推。
4. 当前版本目录内放置 `assets/`、`design/`、`提示词.md`、`poster.html`、`dist/`。
5. 启用子 agent 在根目录 `assets/` 中选择合适素材，使用 `$imagegen` 设计 1 张 UI 效果图，保存到当前版本 `design/`。
6. 启用子 agent 按设计稿获取选中的素材，复制到当前版本 `assets/`。
7. 使用 `$frontend-design` 完成 HTML + CSS 开发，生成 `poster.html`，并按设计稿完成排版。
8. 启用子 agent 使用 `$agent-browser` 打开本地 HTML、截图调试，并导出最终图片到 `dist/`。
9. 子 agent 按还原度、构图、民宿感、文字层级和最终截图质量评分。
10. 主线程每次只针对一个主要问题迭代 HTML 或设计提示词。
11. 完成一次开发后，按根目录 `COMMIT_CONVENTION.md` 进行 git 提交并推送。

## HTML 海报

- 图文介绍类海报优先使用 HTML + CSS 实现。
- 当前版本目录内使用 `poster.html` 开发排版，截图输出到 `dist/`。
- HTML 适用场景：房型介绍、周边介绍、多图排版、文字层级、卡片、渐变、响应式微调。
- 默认字体使用 `PingFangSC`，不可用时使用系统默认字体。
- 开发完成后必须由子 agent 使用 `$agent-browser` 打开本地 HTML、截图调试并导出最终图片。
- 默认截图尺寸为竖版，优先保证美观度，不限制高度和大小。

## 约束

- 每轮输出必须保存到当前版本目录的 `dist/`。
- 方向确认正确后，才考虑生成多张变体。
- 不覆盖旧图，使用递增文件名。
- 只在用户确认后进入下一轮生成。
- 每次提交只包含一个可独立理解、可独立回滚的变更。
