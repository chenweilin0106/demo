# 民宿海报生成流程

## 资源目录

- `assets/周边环境/`：海岸、露台、航拍、建筑外观、周边氛围。
- `assets/客厅/`：客厅、石墙、原木、绿植、公共空间。
- `assets/双床房/`：双床房、窗景、阳台、房间细节。
- `assets/大床房/`：大床房、窗景、卫浴、房间细节。

## 默认流程

1. 先了解用户本轮海报目标、风格、尺寸、用途和限制。
2. 读取 `assets/` 分类，从原始素材中选取合适图片。
3. 新建递增版本目录：已有 `V1` 时新建 `V2`，依次类推。
4. 当前版本目录内放置 `assets/`、`提示词.md`、`dist/`。
5. 用子 agent 只做素材理解、视觉 brief 压缩、生成结果评审。
6. 主线程负责调用 `$imagegen`、保存图片、维护版本目录。
7. 每轮生成 3 张无字主视觉，输出到当前版本的 `dist/`。
8. 子 agent 按真实感、构图、民宿感、可排版空间评分。
9. 主线程保留最佳图，每次只针对一个主要问题迭代提示词。
10. 完成一次开发后，按根目录 `COMMIT_CONVENTION.md` 进行 git 提交并推送。

## 约束

- 默认不让子 agent 直接生成图片。
- 默认先生成无字主视觉，文字后期再加。
- 每轮输出必须保存到当前版本目录的 `dist/`。
- 不覆盖旧图，使用递增文件名。
- 只在用户确认后进入下一轮生成。
- 每次提交只包含一个可独立理解、可独立回滚的变更。
