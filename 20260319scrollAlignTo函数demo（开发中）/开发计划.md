# scrollAlignTo 开发计划（demo 驱动）

## 0. 范围与约束
- 只在 `D:\Project\demo\20260319scrollAlignTo函数demo（开发中）` 开发，不改业务仓库代码。
- demo 为“直接打开 HTML 即可用”的形式：`index.html` 引入 `scrollAlignTo.js`。
- 支持单层滚动容器（element 为 container 后代）；不处理复杂嵌套滚动链路。

## 1. 目标
封装一个可复用滚动对齐函数 `scrollAlignTo`，用于把子元素滚到容器的：
- `start` / `center` / `end`
- 支持横向/纵向
- 可配置是否忽略：父容器 padding、子元素 margin、子元素 border
- 统一处理字符串/数字入参（`'12'`、`12` 都可）

默认行为：
- 忽略父元素 padding（对齐基准默认按 content 区）
- 忽略子元素 margin
- 忽略子元素 border（按 padding box 作为元素对齐盒）

## 2. API 设计（demo 内的 JS 函数）
```js
scrollAlignTo(element, container, {
  direction = 'h',
  align = 'start',
  behavior = 'smooth',
  offset = 0,
  clamp = true,
  ignoreContainerPadding = true,
  ignoreElementMargin = true,
  ignoreElementBorder = true,
  debug = false
})
```

## 3. 算法口径（核心）
- 使用 `getBoundingClientRect()`（视口坐标）计算 `elementRect` 与 `containerRect` 的相对差值。
- 用 `container.scrollLeft/Top` 把“视口差值”转换到“内容坐标系”，避免二次滚动不一致。
- 容器对齐点：可选择是否忽略容器 padding（start/center/end 三种）。
- 元素对齐盒：可选择是否忽略元素 border、是否忽略元素 margin。
- 目标滚动量：`target = elementAlignCoordInScroll - containerAlignPointInViewport + offset`
- `clamp=true` 时限制在 `[0, maxScroll]`。

## 4. demo 设计（覆盖场景）
- 横向/纵向两个容器
- 可调：容器 padding（支持对称/不对称）、元素 margin、元素 border
- 可切：start/center/end，连续点击两次验证稳定性

## 5. 验收标准
- 同一参数配置下，连续点击滚动两次，位置不漂。
- 覆盖：padding/margin/border（对称/不对称）、横纵、start/center/end。
- `offset` 为字符串也能正常生效（不出现 NaN）。
