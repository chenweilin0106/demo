# 周报模板：内容生成规范（基于 Git 提交）

目标：把“本周要合入/要提交的内容”稳定地总结成多级有序列表，并生成：
- 必须：`周报_YYYY_MM_DD_YYYY_MM_DD.html`（用于钉钉粘贴）
- 可选：`周报_YYYY_MM_DD_YYYY_MM_DD.md`（详细周报/留痕回溯用，含统计口径与参考提交）

说明：
- 排版与复制规则以 `周报模板_格式规范.md` 为准（不带字体、无空行、钉钉粘贴编号正确）。

## 1. 输入参数（建议每次都显式提供）

- 时间范围：`YYYY-MM-DD ~ YYYY-MM-DD`（含首尾）
  - 建议不要只说“这周”，避免口径歧义
- 基准分支：默认 `origin/trunk`
- 分支列表：`branch1, branch2, ...`
- 作者：默认 `chenweilin`
- 是否包含 merge：默认不包含（`--no-merges`）

建议（可选但更稳）：
- 给每个分支指定“一级标题名”（映射表），用于强制归类，例如：
  - `weekly_gardenParty_test_20260408 -> 春日活动`
  - `vipDay_branch_dev_20260421 -> 4月会员日`
  - `2025ServerOpening_branch_dev_20251020 -> 开服活动`

## 2. Git 提取口径（固定）

对每个分支，提取“相对 trunk 的待合入差异”范围内、符合作者与时间的提交：

```bash
git log origin/trunk..$branch \
  --no-merges \
  --author="chenweilin" \
  --since="YYYY-MM-DD 00:00:00" \
  --until="YYYY-MM-DD 23:59:59" \
  --date=short \
  --pretty=format:"%h %ad %s"
```

说明：
- 用 `origin/trunk..branch` 是为了只统计“该分支相对 trunk 新增的提交”
- 用 `--since/--until` 是为了只统计本周（避免分支里历史提交过多）

## 3. 去重规则（避免重复工作项）

按下面顺序去重：
1) 跨分支若出现相同 `commit hash`：只保留一次
2) 若不同 hash 但 `subject` 完全相同：合并为一条（保留一次）

## 4. 归类规则（生成 1/a/i）

### 4.1 一级标题（1. / 2. / 3.）
优先使用“分支 -> 一级标题名”的映射；如果没有映射：
- 使用提交 subject 的前缀做粗归类（例如 `春日活动-...` / `开服活动-...`）
- 无法判断时：用分支名或项目名作为兜底

### 4.2 二级标题（a. / b. / c.）
推荐统一成少量固定类别（便于长期可读），按关键词匹配：
- 包含 `接口`：接口对接
- 包含 `资源` / `图片`：资源更新
- 包含 `样式` / `UI`：样式/交互优化
- 包含 `动画` / `动效`：动效优化
- 包含 `修复` / `bug`：问题修复
- 包含 `CDN` / `外网` / `发布`：性能/发布相关
- 其它：其他

若某个一级标题下只有一类内容，二级标题可以只保留 1 个（例如只用 `a.`）。

### 4.3 三级及以上条目（i. / ii. / iii. ...）
三级条目默认作为最终展示的“工作项”；如果某条目需要继续拆分，可以在该条目下继续追加子级列表。

层级与编号（按 `1/a/i` 循环，4 级开始回到 `1.`）：
- 1 级：`1.` `2.` `3.` ...
- 2 级：`a.` `b.` `c.` ...
- 3 级：`i.` `ii.` `iii.` ...
- 4 级：回到 `1.` `2.` `3.` ...
- 5 级：回到 `a.` `b.` `c.` ...
- 6 级：回到 `i.` `ii.` `iii.` ...
- 7 级及以上：继续按 `1/a/i` 循环

文案要求：
- 用短句，尽量是“动词 + 名词/对象”（例如：`新增首页动画`、`进度条样式优化`）
- 避免把同一句“更新/优化”重复写多次：能合并就合并
- 禁止出现分支名、hash、文件路径
- 禁止出现 Markdown 语法（反引号等）
- 英文/数字与中文之间不留空格（详见 `周报模板_格式规范.md`）

示例（出现 4 级时）：

```text
1. 春日活动
   a. 踏春登榜
      i. 奖励修改
         1. 奖励配置校对
         2. 文案调整
```

## 5. 输出文件命名（固定）

- `周报_YYYY_MM_DD_YYYY_MM_DD.html`：最终产物（打开后点击复制，再粘贴到钉钉日志）
- `周报_YYYY_MM_DD_YYYY_MM_DD.md`（可选）：详细周报/留痕回溯（建议保留）
  - 建议包含：统计口径、归类后的工作项、参考提交（hash + subject）

示例参考：`周报_2026_03_23_2026_03_27.md`

## 6. 推荐提示词模板（你后续直接这样说就行）

```text
帮我生成这周的周报：
- 时间范围：YYYY-MM-DD ~ YYYY-MM-DD（含首尾）
- 基准分支：origin/trunk
- 作者：chenweilin
- 分支列表：
  - branch1（可选：映射到一级标题名）
  - branch2（可选：映射到一级标题名）
  - ...
输出要求：
- 按多级有序列表归类总结（默认 1/a/i；3 级以上继续按 1/a/i 循环）
- 生成最终 HTML：周报_YYYY_MM_DD_YYYY_MM_DD.html
- 可选：同时生成详细周报 md：周报_YYYY_MM_DD_YYYY_MM_DD.md（含统计口径与参考提交）
- 不带分支名/hash/文件路径；不使用 Markdown 标记；英文数字与中文不留空格；列表项之间无空行
- 复制粘贴到钉钉日志后：编号不乱、无多余空行、字体跟随钉钉默认
```
