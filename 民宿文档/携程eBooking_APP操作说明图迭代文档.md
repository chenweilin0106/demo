# 携程 eBooking App 操作说明图迭代文档

## 目标

制作一张给老人临时自助操作使用的 App 操作说明图。

这张图不是业务流程图，而是“照着点”的使用说明图。重点是让使用人知道：

- 从哪里打开 App。
- 登录时点哪里。
- 去哪里看订单。
- 如何确认预订。
- 如何确认入住。
- 如何查看待处理、待入住、已入住订单。

## 统一设计要求

- 图片使用竖版长图，适合手机查看，也方便后续打印。
- App 页面使用重新生成的示例 App 截图风格，不直接展示真实订单信息。
- 登录页面不展示真实账号和密码。
- 每一步都使用大号红色点击圆点、红色箭头或红色圈选标注。
- 提示文字要短，尽量使用“点击这里”“点这个”“先看日期”“核对姓名”这类直接表达。
- 不使用复杂说明，不写长段文字。
- 图片生成完成后统一放在当前目录。

## v1：基础操作说明图

v1 先做基础版，不做红色警告区。

### v1 覆盖内容

1. 打开 App
   - 找到“携程 eBooking”图标。
   - 点击打开。

2. 登录 App
   - 点击“输入用户名”。
   - 点击“输入密码”。
   - 勾选同意协议。
   - 点击“登录”。
   - 不展示真实账号和密码。

3. 确认预订
   - 点击底部“订单”。
   - 点击顶部“待处理”。
   - 找到客人订单。
   - 点击订单进入详情。
   - 核对姓名、房型、入住日期、离店日期。
   - 点击“确认预订”或页面实际显示的确认按钮。
   - 回到“待入住”能看到订单，表示确认成功。

4. 确认入住
   - 点击底部“订单”。
   - 点击顶部“待入住”。
   - 选择正确入住日期。
   - 找到客人订单。
   - 点击订单进入详情。
   - 核对姓名、房型、入住日期、离店日期。
   - 点击“确认入住”或页面实际显示的确认按钮。
   - 回到“已入住”能看到订单，表示确认入住成功。

5. 查看订单
   - 查看待处理：订单 → 待处理。
   - 查看未入住：订单 → 待入住 → 选择入住日期。
   - 查看已入住：订单 → 已入住 → 选择入住日期。
   - 页面显示“暂无订单”时，先检查顶部分类和日期是否选对。

### v1 图片建议结构

- 顶部标题：携程 eBooking 操作说明
- 第一段：打开和登录
- 第二段：确认预订
- 第三段：确认入住
- 第四段：查看订单

- 不增加红色警告区，警告区仍放到 v2。

   - 待入住列表重点标注“确认入住”按钮。
   - 所有订单处理都必须先点击底部 Tab 栏的“订单”，再在订单页处理。
   - 参考 `待处理订单空状态_旧版.jpg` 强化“点击底部订单”的视觉路径。

3. 登录页面
   - 参考 `携程eBooking登录页.jpg`。
   - 保留“输入用户名”“输入密码”“登录”“手机验证登录”“微信登录”“邮箱验证登录”等真实页面元素。
   - 不展示账号密码。

4. 查看订单
   - 查看待处理：`订单 → 待处理`。
   - 查看未入住：`订单 → 待入住 → 选日期`。
   - 查看已入住：`订单 → 已入住 → 选日期`。

## v1 高清修正版：统一从订单 Tab 进入

v1 高清修正版根据试用反馈继续调整。

### 本版修正点

- 订单处理不再从首页“待处理订单”进入。
- 确认预订、确认入住、查看订单都统一从底部 Tab 栏点击“订单”进入。
- 图片需要更清晰，优先采用真实截图作为底图，再手工叠加大号中文说明、红色箭头、红色点击圆点。
- 尽量避免让 AI 直接生成整张含大量中文小字的图片，减少糊字、错字和界面漂移。

### 本版主路径

1. 确认预订
   - 打开 App。
   - 点击底部“订单”。
   - 点击顶部“待处理”。
   - 找到客人订单。
   - 进入详情核对姓名、房型、日期。
   - 点击“确认预订”。

2. 确认入住
   - 点击底部“订单”。
   - 点击顶部“待入住”。
   - 选择正确入住日期。
   - 找到客人订单。
   - 点击“确认入住”。

3. 查看订单
   - 查看待处理：订单 → 待处理。
   - 查看待入住：订单 → 待入住 → 选日期。
   - 查看已入住：订单 → 已入住 → 选日期。

## 后续生成图片注意事项

- 优先保证中文文字准确，其次才是画面美观。
- AI 生成的 App 截图只做示意，不使用真实订单号、真实手机号、真实价格和真实客户姓名。
- 如果 AI 生成中文按钮不准确，应后期手动替换文字。
- 红色点击标注要明显，避免老人看不清。
- 每张示例订单中统一使用“张先生”或“示例客人”。
- 房型示例可以使用“标准大床房”“海景双床房”。
