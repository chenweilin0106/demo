# WSL 踩坑笔记

## 1. “装完了但不能用”：很多时候只是没重启
现象：`wsl --status` 仍提示“不支持 WSL2”，或 `wsl -l -v` 显示没有发行版。

优先做两件事：
- 重启 Windows
- 再跑：`wsl --status`、`wsl -l -v`

更完整排错：`WSL2-安装.md`

## 2. `E_ACCESSDENIED` / 输出乱码
现象示例：`Wsl/EnumerateDistros/Service/E_ACCESSDENIED`。

常用处理（管理员 PowerShell，会中断当前 WSL 实例）：

```powershell
Stop-Service -Name WSLService -Force
Start-Service -Name WSLService
wsl --shutdown
wsl -l -v
```

## 3. `/mnt/c` / `/mnt/d` 运行很慢
- Windows 盘可以读写，但遇到“依赖安装/构建/大量小文件”时经常变慢。
- 思路：把仓库放到 Linux 路径（如 `~/projects`），在 WSL 里运行。

## 4. SSH 两类“指纹”，不要混了
### 4.1 服务器指纹（Host key）
第一次连 `github.com`/`gitlab.com` 时提示：
`The authenticity of host ... can't be established`。

这问的是“服务器是谁”，需要把 host key 写进 `~/.ssh/known_hosts`。

### 4.2 账号公钥（Authentication key）
如果报：`Permission denied (publickey)`，通常是：
- 你用的那把 key 的“公钥”没有加到平台账号里（GitHub/GitLab 的 SSH Keys）
- 或者你实际使用的 key 和你以为的不一样（常见：Windows 能用，但 WSL 用了另一把）

排查建议：
```bash
ssh -v -T git@github.com
# 看它在 Offering 哪一把 key
```

## 5. Windows/WSL 共用密钥的权限坑
- 直接从 WSL 使用 `/mnt/c/Users/<user>/.ssh/id_xxx`，有时会因为权限过宽而被 OpenSSH 拒绝。
- 更稳妥：复制一份私钥到 WSL 的 `~/.ssh/`，并 `chmod 600`。

## 6. 代理端口连不上（Clash Verge）
典型原因：Windows 侧代理只监听 `127.0.0.1`，WSL2 访问不到。

思路：开启 `Allow LAN/允许局域网`，并确认端口监听在 `0.0.0.0`。

完整配置与自检：`WSL2-安装.md`

## 7. `networkingMode=mirrored` + Clash Fake-IP：WSL “看起来断网”
背景：把 WSL2 网络从 NAT 改成 mirrored（`%USERPROFILE%\\.wslconfig` → `[wsl2] networkingMode=mirrored`）。

现象：
- `ping 1.1.1.1` 正常，但 `curl https://example.com`/`curl https://github.com` 超时
- `getent hosts example.com` 解析到 `198.18.0.x`（fake-ip 段）

根因：
- Clash Verge / mihomo 的 DNS 使用 `fake-ip`，把域名解析成 `198.18.0.0/16`
- 在 mirrored 下，WSL 的流量没有被 Windows 侧透明代理/接管时，就会直连 fake-ip → 访问失败 → 误以为“没网”

快速自检：
```bash
env | grep -i proxy
getent hosts example.com
curl -I -m 8 https://example.com
curl -I -m 8 -x http://127.0.0.1:7897 https://example.com
```

解决（本次采用：让 WSL 主动走 Windows 本机代理）：
- `~/.bashrc`：提供 `proxy_on/proxy_off/proxy_status`，并默认 `proxy_on`；设置 `http_proxy/https_proxy/no_proxy`（含大写版本）
- `~/.profile`：补充 `http_proxy/https_proxy`（保证 login shell 也生效）
- 端口：`7897` 为 Clash Verge 的 mixed port，按实际端口调整

备注：
- `no_proxy` 建议至少包含 `localhost,127.0.0.1,::1`
- 某些站点 `curl -I` 返回 `403 cf-mitigated: challenge` 属于 Cloudflare 挑战，不等于网络不通
