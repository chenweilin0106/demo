# WSL 安装和使用笔记

> 目标：把常用项目放到 WSL 的 Linux 文件系统里（如 `~/projects`），在 WSL 里开发/构建更快、更顺。

## 1. 快速判断你现在的状态
在 Windows PowerShell 执行：

```powershell
wsl --status
wsl -l -v
```

- 期望：能看到发行版（常见 Ubuntu），且 `VERSION` 为 `2`。

进入 Ubuntu：

```powershell
wsl -d Ubuntu
```

## 2. 安装/初始化 WSL2（首次安装通常需要重启）
管理员 PowerShell：

```powershell
wsl.exe --install -d Ubuntu --web-download
```

如果 `--web-download` 不支持，回退为：

```powershell
wsl.exe --install -d Ubuntu
```

安装完成后按提示重启。

更完整的安装、迁移发行版、代理与排错：

## 3. 把项目放到 WSL 里（性能优化的关键）
建议把“经常构建/依赖安装”的仓库放到 Linux 路径（例如）：

```bash
mkdir -p ~/projects
cd ~/projects
```

两种做法：

### 3.1 直接在 WSL 里 `git clone`（推荐）
```bash
cd ~/projects
git clone <repo-ssh-url>
```

### 3.2 从 Windows 盘拷贝过去（可选）
如果你要把 `D:\Project\demo` 这种目录搬过去：

```bash
rsync -a --info=progress2 /mnt/d/Project/demo/ ~/projects/demo/
```

## 4. Git/SSH：在 WSL 里用同一把密钥（与 Windows 共用）
核心原则：
- “共用”指 **同一对密钥**（同一个 `id_rsa` / `id_ed25519`），不代表一定要直接用 `/mnt/c/...` 的文件。
- 为了避免权限/兼容问题，通常建议把 Windows 的私钥复制一份到 WSL 的 `~/.ssh/`，并设置权限。

本仓库这次验证通过的配置方式（示例：强制 GitHub 用 `id_rsa`）：

```bash
# 复制密钥（示例用户名 dcw220）
mkdir -p ~/.ssh && chmod 700 ~/.ssh
install -m 600 /mnt/c/Users/dcw220/.ssh/id_rsa ~/.ssh/id_rsa
install -m 644 /mnt/c/Users/dcw220/.ssh/id_rsa.pub ~/.ssh/id_rsa.pub

# 强制 github.com 使用该密钥
cat > ~/.ssh/config <<'EOF'
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_rsa
  IdentitiesOnly yes
EOF
chmod 600 ~/.ssh/config

ssh -T git@github.com
```
