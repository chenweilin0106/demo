# Git 使用笔记（常用命令）

## 1. 仓库基础

```bash
git init
git clone <repo_url>

git status
git add <file>
git add -A
git commit -m "feat: xxx"
```

## 2. 差异与日志

```bash
git diff
git diff --staged

git log --oneline --decorate --graph --all
git show <commit>
git blame <file>
git grep "<keyword>"
```

## 3. 分支

```bash
git branch
git branch -a

git switch -c feature/xxx
git switch main

git merge <branch>
git rebase <branch>
git rebase -i HEAD~3

git branch -m <old> <new>   # 重命名分支
git branch -d <branch>      # 安全删除（需已合并）
git branch -D <branch>      # 强制删除
```

## 4. 远端与同步

```bash
git remote -v
git remote add origin <repo_url>

git fetch --all --prune
git pull
git pull --rebase

git push
git push -u origin <branch>
git push -u origin HEAD

git push origin --delete <branch>    # 删除远端分支
git push --force-with-lease          # 谨慎使用：更安全的强推
```

## 5. 撤销/回滚（高频）

```bash
git restore <file>                   # 丢弃工作区修改
git restore .                        # 丢弃全部工作区修改
git restore --staged <file>          # 把暂存区撤回到工作区

git reset --soft HEAD~1              # 回退 1 次提交（保留暂存区）
git reset --mixed HEAD~1             # 回退 1 次提交（保留工作区）
git reset --hard HEAD~1              # 回退 1 次提交（丢弃一切修改，危险）

git revert <commit>                  # 生成“反向提交”，适合已推到远端的场景
git reflog                           # 找回“丢失的提交/分支”
```

## 6. 暂存区与提交小技巧

```bash
git diff --staged
git commit --amend                   # 修改最近一次提交（内容/说明）
```

## 7. Stash（临时收纳）

```bash
git stash
git stash -u                         # 包含未跟踪文件
git stash list
git stash pop
git stash apply stash@{0}
```

## 8. Tag（打版本）

```bash
git tag
git tag -a v1.0.0 -m "v1.0.0"
git push origin v1.0.0
git push --tags
```

## 9. Submodule（子模块）

```bash
git submodule add <repo_url> <path>
git submodule update --init --recursive
git submodule update --remote --merge
```

## 10. 清理（谨慎）

```bash
git clean -nd                        # 预览会删什么
git clean -fd                        # 删除未跟踪文件/目录
git clean -fdx                       # 连同被 .gitignore 忽略的也删（非常危险）
```
