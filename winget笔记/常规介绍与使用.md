# winget 常规介绍与使用

## 1. winget 是什么

`winget`（Windows Package Manager CLI）是 Windows 上的命令行包管理工具，用来搜索、安装、升级、卸载软件，并可导出/导入软件清单。

## 2. 快速自检

```powershell
winget --version
winget --info
winget source list
```

常见源：
- `winget`：社区仓库（多数常用软件）
- `msstore`：Microsoft Store（可能受登录/地区/策略影响）

## 3. 常用命令速查

### 3.1 搜索与查看详情

```powershell
winget search docker
winget show -e --id Docker.DockerDesktop
```

建议优先用 `--id` + `-e`（精确匹配），避免同名软件装错。

### 3.2 安装

```powershell
winget install -e --id Docker.DockerDesktop --accept-package-agreements --accept-source-agreements
```

常用参数：
- `--silent`：尽量静默安装（是否生效取决于安装器）
- `--interactive`：交互式安装（需要确认时用）
- `--scope user|machine`：安装范围（部分软件支持）
- `--source winget|msstore`：指定来源

### 3.3 列出已安装软件

```powershell
winget list
winget list docker
```

### 3.4 升级

```powershell
winget upgrade
winget upgrade -e --id Docker.DockerDesktop --accept-package-agreements --accept-source-agreements
```

升级全部（谨慎）：
```powershell
winget upgrade --all --accept-package-agreements --accept-source-agreements
```

### 3.5 卸载

```powershell
winget uninstall -e --id Docker.DockerDesktop
```

### 3.6 管理源

```powershell
winget source update
winget source reset --force
```

当搜索/升级结果异常时，优先 `source update`，再考虑 `source reset --force`。

### 3.7 导出 / 导入（装机/迁移）

导出：
```powershell
winget export -o .\\winget-export.json
```

导入：
```powershell
winget import -i .\\winget-export.json --accept-package-agreements --accept-source-agreements
```

## 4. 常用场景示例

### 4.1 安装/重装 Docker Desktop（更稳的做法）

```powershell
winget install -e --id Docker.DockerDesktop --accept-package-agreements --accept-source-agreements
```

如果 Docker Desktop 应用内提示更新，但 `winget upgrade` 看不到新版，优先走“官方全量安装器”升级，详见 `踩坑笔记.md`。

### 4.2 生成安装日志（排错）

部分命令支持 `--log`（以实际 `winget` 版本为准）：
```powershell
winget install -e --id Docker.DockerDesktop --log .\\winget-install.log
```

## 5. 使用建议

- 优先使用 `--id`（配合 `-e`）安装/卸载。
- 安装后如果命令行里找不到新装的可执行文件，通常是 **PATH 需要新开一个终端** 才会生效。
- 能升级就用 `winget upgrade`；遇到应用内“增量更新”容易翻车的产品，优先用 **全量安装器** 或 `winget install/upgrade`。
