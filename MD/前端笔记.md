12. 记录：Webpack 5 图片资源处理配置

    #### 功能描述
    Webpack 5 内置了对资源模块的支持，可以根据文件大小自动选择是将图片转成 `dataUrl` 还是使用本地资源的地址。

    #### 使用的 Loader
    在 Webpack 5 中，不再需要显式配置 `url-loader` 或 `file-loader`，而是使用内置的资源模块处理功能。

    #### 配置步骤
    1. 在 `vue.config.js` 文件中，使用 `chainWebpack` 方法添加一个新的规则来处理图片资源。
    2. 使用 `type: 'asset'` 来自动选择 base64 编码或资源文件。
    3. 配置 `parser.dataUrlCondition.maxSize` 来设置将图片转成 `dataUrl` 的大小阈值。

    #### 示例代码
    ```javascript
    module.exports = {
      chainWebpack(config) {
        config.module
          .rule('images')
          .test(/\.(png|jpe?g|gif|svg)$/)
          .type('asset')
          .parser({
            dataUrlCondition: {
              maxSize: 10240 // 小于 10KB 的图片将被转成 dataUrl
            }
          })
      }
    }
    ```

    #### 说明
    - `type: 'asset'`：自动选择 base64 编码或资源文件。
    - `parser.dataUrlCondition.maxSize`：设置将图片转成 `dataUrl` 的大小阈值，单位为字节。

15. ### 问题与解答

    #### 问题 1
    **用户**: 那就是调用系统API通知浏览器并传递IP地址，你不用给出示例代码，我能够理解，不理解我会和你沟通的

    **解答**: 好的，操作系统通过系统API（如`getaddrinfo`）将解析出的IP地址返回给浏览器。浏览器调用该API时会阻塞，直到操作系统返回结果。操作系统接收到DNS服务器的响应后，解析出IP地址并通过系统调用的返回值传递给浏览器。

    #### 问题 2
    **用户**: 浏览器调用该API时会阻塞，这句话什么意思，再DNS没有返回IP地址之前，浏览器的这个标签页啥也干不了是嘛

    **解答**: 是的，浏览器调用系统API（如`getaddrinfo`）进行DNS解析时会阻塞，意味着在DNS解析完成并返回IP地址之前，浏览器的这个标签页无法继续进行后续的网络请求操作。这个阻塞只影响当前的网络请求，不会影响整个浏览器或其他标签页的操作。

    #### 问题 3
    **用户**: 那这里又有一个疑问了，浏览器怎么知道我哪个标签页需要这个IP地址，我开了5个标签页，浏览器怎么知道是哪一个

    **解答**: 浏览器通过维护每个标签页的上下文来区分不同的网络请求。每个标签页都有自己的独立上下文，包括网络请求队列、缓存等。当一个标签页发起DNS解析请求时，浏览器会将该请求与标签页的上下文关联起来。以下是简要的过程：

    1. **标签页发起请求**：
       - 每个标签页都有自己的上下文和网络请求队列。
       - 当标签页发起DNS解析请求时，浏览器会将该请求与标签页的上下文关联。

    2. **操作系统处理请求**：
       - 操作系统接收到DNS解析请求后，进行处理并返回结果。

    3. **浏览器接收结果**：
       - 浏览器接收到操作系统返回的IP地址后，会根据请求的上下文将结果返回给发起请求的标签页。

    #### 总结
    - 浏览器通过维护每个标签页的上下文来区分不同的网络请求。
    - 每个DNS解析请求都会与发起请求的标签页上下文关联，确保结果返回给正确的标签页。

    #### 问题 4
    **用户**: 也就是每个标签发起的请求都会带有一个代表这个标签的id，用来标识，方便后面对接系统返回的IP地址

    **解答**: 是的，每个标签页发起的请求都会带有一个唯一的标识符（如标签页ID），用于区分不同标签页的请求，并确保系统返回的IP地址能够正确地对应到发起请求的标签页。

17. vue2,vue-cli 5 开发：提高热更新速度
```javascript
// npm install -D cache-loader thread-loader
module.exports = {
  productionSourceMap: false,
  
  devServer: {
    hot: true,
    compress: true,
    client: {
      overlay: false
    },
    static: {
      watch: {
        ignored: [/node_modules/, /public/],
        poll: false
      }
    }
  },

  configureWebpack: {
    performance: { hints: false },
    cache: {
      type: 'filesystem',
      buildDependencies: {
        config: [__filename]
      },
      allowCollectingMemory: true
    }
  },

  chainWebpack(config) {
    // 添加缓存和多线程加载
    config.module
      .rule('vue')
      .use('cache-loader')
      .loader('cache-loader')
      .options({
        cacheDirectory: path.resolve(__dirname, 'node_modules/.cache/vue-loader')
      })
      .end()
      .use('thread-loader')
      .loader('thread-loader')
      .end()

    // 开启babel-loader缓存
    config.module
      .rule('js')
      .use('babel-loader')
      .loader('babel-loader')
      .options({
        cacheDirectory: true
      })

    config.plugins.delete('prefetch')

    // 优化监视选项
    config.watchOptions({
      ignored: ['node_modules/**', 'public/**', 'dist/**']
    })
  }
}
```
