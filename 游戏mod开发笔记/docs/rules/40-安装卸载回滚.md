## 1. 安装清单

`manifest.json` 必须作为安装计划来源。

`manifest.json` 必须放在开发目录根目录。

发布包内必须包含当前版本的 `manifest.json`。

`manifest.json` 固定字段：

```json
{
  "modName": "",
  "version": "",
  "gameName": "",
  "installTarget": "",
  "backupDir": "",
  "installRecord": "",
  "files": [
    {
      "source": "",
      "target": "",
      "sha256": "",
      "backup": true
    }
  ]
}
```

每次变更 Mod 文件后，必须更新 `manifest.json`。

初始化模板中的 `files: []` 只是占位，新增 Mod 文件后必须按固定字段结构填写文件对象。

字段语义：

- `installTarget`：相对游戏根目录的安装目标目录。
- `backupDir`：相对安装目标目录的安装期备份目录，固定为 `_mod_backups/{Mod名称}/{版本号}/`。
- `installRecord`：相对安装目标目录的安装记录路径，固定为 `_mod_records/{Mod名称}-{版本号}.json`。

`manifest.json` 禁止包含本机绝对路径。

## 2. 安装、卸载、回滚

安装记录必须作为卸载和回滚依据。

回滚必须依赖安装记录和备份文件。

禁止靠记忆手动猜测回滚。

开发期备份和安装期备份必须分开。

开发期备份固定放在开发目录：

```text
backups/
```

安装期备份固定放在安装目标目录下：

```text
_mod_backups/{Mod名称}/{版本号}/
```

安装记录固定放在安装目标目录下：

```text
_mod_records/{Mod名称}-{版本号}.json
```

安装记录必须包含：

- Mod 名称。
- Mod 版本号。
- 游戏根目录。
- 安装目标目录。
- 已安装文件。
- 备份文件。
- 安装前文件 `sha256`。
- 安装后文件 `sha256`。

安装脚本必须做到：

1. 接收游戏根目录参数。
2. 检查游戏根目录。
3. 检查当前 Mod 版本号。
4. 读取 `manifest.json`。
5. 检查安装目标目录。
6. 校验 Mod 文件 `sha256`。
7. 检查已安装版本。
8. 检查目标文件冲突。
9. 备份即将覆盖的文件。
10. 复制 Mod 文件。
11. 写入安装记录。
12. 输出中文安装结果。

安装脚本禁止硬编码用户本机路径。

安装脚本必须支持 `--dry-run`，用于只检查不写入。

安装脚本参数固定格式：

```text
install --game-dir {游戏根目录}
install --game-dir {游戏根目录} --dry-run
```

目标文件已存在且不属于当前 Mod 安装记录时，必须暂停安装。

目标文件已被其他 Mod 或人工修改时，必须暂停安装。

安装成功提示固定格式：

```text
安装完成：{Mod名称} {版本号}
游戏根目录：{游戏根目录}
安装目标目录：{安装目标目录}
```

卸载脚本必须做到：

1. 接收游戏根目录参数。
2. 读取安装记录。
3. 校验当前已安装 Mod 名称和版本号。
4. 删除 Mod 文件。
5. 恢复备份文件。
6. 删除安装记录。
7. 输出中文卸载结果。

卸载脚本禁止硬编码用户本机路径。

卸载脚本必须支持 `--dry-run`，用于只检查不写入。

卸载脚本参数固定格式：

```text
uninstall --game-dir {游戏根目录}
uninstall --game-dir {游戏根目录} --dry-run
```

卸载成功提示固定格式：

```text
卸载完成：{Mod名称} {版本号}
原文件已恢复。
```

每次安装后必须能回答：

- 安装了哪些文件。
- 覆盖了哪些文件。
- 备份保存在哪里。
- 如何卸载。
- 如何恢复原文件。

回滚验证必须覆盖：

- 安装失败后的自动回滚。
- 卸载后的原文件恢复。
- 升级失败后的旧版本恢复。

## 3. 版本升级

安装前必须检查当前游戏根目录是否已安装同名 Mod。

已安装同版本时，必须暂停安装。

已安装旧版本时，必须执行升级流程。

升级流程固定为：

1. 读取旧版本安装记录。
2. 记录旧版本号。
3. 卸载旧版本或备份旧版本覆盖文件。
4. 安装新版本。
5. 写入新版本安装记录。
6. 记录升级结果。

升级失败时，必须按旧版本安装记录回滚。

禁止直接覆盖未知版本。
