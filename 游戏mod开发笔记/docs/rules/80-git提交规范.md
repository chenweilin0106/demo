# Git 提交规范

## 1. 拆分原则

- 一次提交只包含一个独立目标。
- 文档、脚本、Mod 文件、打包产物、修复尽量分开提交。
- 不把未验证修改和已验证修改混在同一次提交。
- 提交前必须检查 `git status` 和 `git diff`。
- Git 根目录必须等于 Mod 开发目录；不一致时禁止提交。
- 创建 Git commit 前必须获得用户明确确认。

## 2. 提交消息格式

```text
{版本号}: {类型}: {摘要}
```

类型：

- `init`：项目初始化。
- `docs`：文档或知识库。
- `feat`：Mod 功能。
- `fix`：问题修复。
- `test`：测试或验证。
- `package`：打包和发布。
- `release`：最终版本交付。
- `chore`：非功能维护。

示例：

```text
v1: init: 初始化 Mod 开发目录
v1: docs: 记录游戏目录和安装目标
v1: feat: 添加基础配置覆盖文件
v1: fix: 修复安装目标冲突检测
v1: package: 生成并验证发布包
```

最终版本提交使用：

```text
{版本号}: release: {版本目标}
```
