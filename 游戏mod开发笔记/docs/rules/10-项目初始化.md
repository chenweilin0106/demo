## 1. 项目初始化

新建 Mod 项目时，必须先创建开发目录。

初始化时必须创建：

- `docs/`
- `knowledge/`
- `external-mods/`
- `scripts/`
- `mod-files/`
- `package/`
- `backups/`
- `releases/`
- `manifest.json`
- `.gitignore`

初始化时必须确认并记录：

- 开发目录。
- 游戏根目录。
- 安装目标目录。
- Mod 名称。
- 初始版本号。
- 游戏名称和游戏版本。
- Mod 加载器名称和版本。
- Git 状态。
- 可用资料来源。
- 可用工具。

未接入 Git 时，必须先初始化 Git。

无法接入 Git 时，必须暂停并说明原因。

## 2. 目录规范

Mod 项目必须使用固定目录结构。

```text
某游戏-mod-dev/
docs/              开发文档
knowledge/         知识库
external-mods/     参考 Mod
scripts/           安装、卸载、打包、测试脚本
mod-files/         Mod 源文件
package/           当前安装包
backups/           开发期备份
releases/          已发布版本
manifest.json      当前版本安装清单
.gitignore         Git 忽略规则
```

目录规则：

- `docs/` 存放 `v1`、`v2`、`v3` 等版本开发文档。
- `knowledge/` 存放字段、文件结构、错误、工具、结论。
- `external-mods/` 存放公开渠道获取的参考 Mod。
- `scripts/` 存放自动化脚本。
- `mod-files/` 存放当前 Mod 文件。
- `package/` 存放当前版本安装包，必须包含当前版本的 `manifest.json`。
- `backups/` 只存放开发期备份，不进入发布包。
- `releases/` 存放已确认的版本产物。
- `manifest.json` 记录当前版本安装计划。

游戏根目录只允许被安装、卸载、测试脚本按清单读写。

禁止把开发文档、知识库、参考 Mod、Git 仓库放入游戏根目录。
