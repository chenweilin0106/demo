## 1. 知识库规则

每次版本开发完成后，必须更新知识库。

必须记录：

- 本版本新增结论。
- 已确认字段。
- 已确认文件用途。
- 崩溃问题。
- 解决方案。
- 无效尝试。
- 工具使用经验。
- 安装、卸载、回滚问题。
- 参考 Mod 结论。
- 修复记录。

问题记录模板：

```md
## 问题

版本：

现象：

触发条件：

相关文件：

最近修改：

排查过程：

解决方案：

是否复现：

是否解决：
```

字段记录模板：

```md
## 字段

版本：

文件：

字段名：

推测含义：

已验证取值：

错误取值表现：

结论：
```

工具记录模板：

```md
## 工具

工具名称：

版本：

来源：

用途：

使用命令：

验证结果：

备注：
```

参考 Mod 记录模板：

```md
## 参考 Mod

名称：

版本：

来源：

文件结构：

安装方式：

能复用的结论：

不能复用的内容：
```

## 2. AGENTS.md 检查规则

每次版本完成后，必须检查当前项目 `AGENTS.md` 是否需要更新。

需要更新的情况：

- 发现新的固定流程。
- 发现新的禁止操作。
- 发现新的安装规则。
- 发现新的测试规则。
- 发现新的目录约定。
- 发现新的工具使用规则。

项目没有 `AGENTS.md` 时，必须在开发文档中记录：

```text
AGENTS.md 检查结果：当前项目不存在，使用上级或全局规则。
```

不需要更新时，必须在开发文档中记录：

```text
AGENTS.md 检查结果：无需更新。
```
