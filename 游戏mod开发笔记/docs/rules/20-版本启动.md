## 1. 版本启动

每次开始开发前，AI 必须由 `game-mod-dev` skill 决定读取范围，并确认：

- 当前游戏 Mod 项目目录结构。
- 当前项目已有文档。
- 当前项目 `AGENTS.md`。
- 当前项目知识库相关内容。
- 当前版本开发文档。
- 当前任务相关的安装脚本、卸载脚本、打包脚本、测试脚本。
- 当前任务相关的参考 Mod 和参考资料。
- 当前游戏版本、Mod 加载器版本、系统环境。
- 当前版本目标、验收标准、安装卸载回滚方案。

当前项目没有 `AGENTS.md` 时，必须记录：

```text
AGENTS.md：当前项目不存在，使用上级或全局规则。
```

当前版本开发文档不存在时，必须先创建并确认版本目标和验收标准。

核心资料缺失时必须暂停：

- 游戏根目录未知。
- 游戏版本未知。
- 当前版本开发文档无法创建。
- 安装方式未知。
- 卸载方式未知。
- 回滚方式未知。
- 验收标准未知。

非核心资料缺失时必须记录缺失项，然后继续：

- 日志位置未知。
- 参考 Mod 不存在。
- 社区资料不足。
- 工具版本未确认。

禁止在没有读取上下文的情况下直接修改文件。

## 2. 版本文档和 Git

每次开发必须有明确版本号，例如：

- `v1`
- `v2`
- `v3`

必须同时使用开发文档和 Git。

开发文档负责记录：

- 本版本目标。
- 验收标准。
- 环境基线。
- 安装、卸载、回滚方案。
- 修改范围。
- 已验证内容。
- 未解决问题。
- 失败尝试。
- 知识库更新记录。
- `AGENTS.md` 检查结果。
- 版本记录。
- 最终结论。

版本开发文档固定命名：

```text
docs/{版本号}.md
```

版本开发文档固定模板：

```md
# {版本号} 开发文档

## 版本目标

## 验收标准

## 环境基线

## 开发目录

## 游戏根目录

## 安装目标目录

## 合规边界

## 安装卸载回滚方案

## 修改范围

## 已验证内容

## 未解决问题

## 失败尝试

## 测试列表

## 知识库更新记录

## AGENTS.md 检查结果

## 版本记录

## 最终结论
```

Git 负责：

- 文件变更记录。
- 差异对比。
- 回滚。
- 分支实验。
- 版本提交。

每个版本完成交付检查后，必须产生一次 Git commit。

提交消息格式以 `docs/rules/80-git提交规范.md` 为准。

开发文档和 Git 不互相替代。
