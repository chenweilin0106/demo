# 游戏 Mod 开发流程索引

默认不要全文读取规则文件。

先列出 `docs/rules/`，再按当前任务读取对应文件。

## 规则文件

- `docs/rules/00-总流程.md`：总流程、术语、核心原则。
- `docs/rules/10-项目初始化.md`：新建开发目录和固定目录结构。
- `docs/rules/20-版本启动.md`：版本启动、版本文档和 Git 基本要求。
- `docs/rules/30-资料收集和工具选择.md`：资料来源、合规边界、Mod 类型、工具选择、无源码规则。
- `docs/rules/40-安装卸载回滚.md`：manifest、安装、卸载、回滚、升级。
- `docs/rules/50-调试和问题修复.md`：日志、观测、开发循环、root cause 修复闭环。
- `docs/rules/60-测试发布交付.md`：测试列表、发布包、交付检查。
- `docs/rules/70-知识库和AGENTS.md`：知识库记录和 AGENTS.md 检查。
- `docs/rules/80-git提交规范.md`：Git 提交拆分和消息格式。

## 常用入口

- 开始新项目：读 `00-总流程.md`、`10-项目初始化.md`。
- 开始新版本：读 `20-版本启动.md`。
- 查资料或选工具：读 `30-资料收集和工具选择.md`。
- 写安装脚本、卸载脚本、回滚方案：读 `40-安装卸载回滚.md`。
- 出错、崩溃、Mod 无效：读 `50-调试和问题修复.md`。
- 测试、打包、发布、交付：读 `60-测试发布交付.md`。
- 更新知识库或 AGENTS.md：读 `70-知识库和AGENTS.md`。
- 准备提交：读 `80-git提交规范.md`。
